Why it didn't pick up When you push files to GitHub, the .github folder is a hidden folder (starts with a dot). Some tools — Windows Explorer, Mac Finder, and even some Git GUI apps — either don't show it or don't include it in a drag-and-drop push. The file must land at exactly this path inside your repo: your-repo-name/ └── .github/ └── workflows/ └── scrape.yml If it's anywhere else (like just github/workflows/scrape.yml without the dot, or scrape.yml in the root), GitHub ignores it entirely.
